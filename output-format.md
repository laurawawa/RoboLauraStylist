# Weekly report format — the contract every scan must follow

Save each run to `reports/YYYY-MM-DD-scan.md` and (once email is wired) send the
same content to laurastrauss@gmail.com. Keep it concise and skimmable. Numbered
items, not bullets, so Laura can reply "yes 2, no 4" easily.

---

## Template

```
# Wardrobe Scan — YYYY-MM-DD

## Source check (inspiration layer)
- One line per source: what's new + whether it shifts the filter.
- Explicitly flag any source that could NOT be read this week (don't fake it).

## Applied filters this run
One line summarizing the active filters (size, fabrics, current category
emphasis, exclusions) — pulled from personal-style-guide.md + recent scan-log.md
feedback.

## Watchlist check
One line per active item in `watchlist.md`: matched (with verified link + offer
number) or "no match this run." Put any match at the TOP of the shortlist.

## Shortlist (up to 10, all verified live this run)
For each item:
N. **Brand + item, size — $price** ([listing](full URL))
   - *Why:* how it fits the guide + which stylist principle it satisfies.
   - *Verify:* what to confirm before buying (origin/fabric %, measurements, etc.).

## Worth a click, not yet page-verified
Items seen in search results but not individually opened. Clearly labeled as
unverified.

## Passed / rejected this run (for the log)
Short list of near-misses and why (sold, off-palette, synthetic, wrong size,
ambiguous listing). This trains future runs.

## Feedback prompt
"Reply with yes/no per number, plus any direction changes."
```

---

## Non-negotiable rules
1. **Every shortlisted listing must be individually opened and confirmed live**
   this run — available for purchase, Buy Now present, seller recently active.
   Stale/sold links are the #1 past failure; reject anything ambiguous.
2. **Full clickable URLs**, not search pages or category pages.
3. **No invented listings, prices, or details.** If you can't verify it, it goes
   under "worth a click, not yet page-verified" or gets dropped.
4. Respect price cap (<$300 unless exceptional) and all style-guide exclusions.
5. Keep the rejected list — it's how the agent learns.
